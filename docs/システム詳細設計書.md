## 2. モジュール設計

### 2.1. ファイル監視クローラー (`crawler/pc_crawler.py`)
*   **概要**: 監視フォルダ（`/watch`）のファイルの作成・編集・削除を監視し、全文検索データベース（`knowledge.db`）のインデックスを最新に保ちます。
*   **特徴**:
    *   `watchdog.observers.Observer` によるリアルタイムイベント検知。
    *   マウントされたファイルサーバーや一部OSでの検知漏れを防ぐため、定期的なスレッドポーリング機能（ポーリング間隔は環境変数 `POLL_INTERVAL_SECONDS` で変更可能）を併用。
    *   ファイルの削除時は、FTS5データベース内の該当レコードを即時にクレンジングする自動クリーンアップロジックを搭載。

### 2.2. RAGエンジン (`app/api/services/rag_engine.py`)
*   **概要**: 質問に対して `knowledge.db` から全文検索（FTS5）で関連箇所を特定し、ローカルLLMへのプロンプトにコンテキストとして注入することで、閉域環境下で高精度な「出典付き回答」を生成します。
*   **特徴**:
    *   Ollama未起動時のスタティック情報による「極上フォールバック回答」を搭載。
    *   PDF財務レポートの即時解析・描画API（`POST /api/chat/analyze-pdf`）を含み、Chart.jsを使用した損益ダッシュボードをUI上で自動構築します。

### 2.3. 音声認識モジュール (`app/api/routers/voice.py`)
*   **概要**: Whisperの軽量モデル（`tiny`）をローカルでロードし、音声入力ファイルを高速にテキスト変換します。
*   **特徴**:
    *   ローカルCPU環境におけるロードおよび変換時間を削減するため、デフォルトで `tiny` モデルを使用。
    *   一時ファイルを安全に削除するクリーンアップロジックを搭載。

### 2.4. 自然言語DB連携（NL2SQL）エンジン (`step2/nl2sql/nl2sql_engine.py`)
*   **概要**: 業務定義カタログをベースに自然言語質問から安全なSELECTクエリを生成・実行します。
*   **特徴**:
    *   大文字小文字を無視した禁止SQL（DROP、UPDATE、DELETEなど）のチェック機能。
    *   セミコロンを使用した複数文実行の遮断。
    *   SQLiteの `?mode=ro`（物理読み取り専用モード）によるDB破壊の完全防止。
    *   LIMIT句の自動付与（最大100件）によるリソース枯渇の防止。

## 3. APIエンドポイント一覧

### 3.1. チャット / RAG関連
*   `POST /api/chat/` : チャット問い合わせ（RAG回答）
*   `POST /api/chat/stream` : ストリーミングチャット
*   `POST /api/chat/analyze-pdf` : PDF財務資料の解析・レポート・ダッシュボード自動描画

### 3.2. データベース連携（NL2SQL）
*   `POST /api/db/query` : 自然言語質問によるSQL自動生成・安全実行
*   `GET /api/db/audit-logs` : データベース監査ログの取得（最大50件）

### 3.3. その他
*   `POST /api/voice/transcribe` : 音声ファイルのテキスト書き起こし
*   `POST /api/documents/generate` : RAG回答に基づく Word / Excel / PPT ドキュメントドラフトの自動生成・ダウンロード
*   `GET /health` : APIヘルスチェック

## 4. データベース設計

### 4.1. 全文検索データベース (`knowledge.db` - SQLite FTS5)
*   **テーブル: `documents` (Virtual Table)**
    *   `file_path` (TEXT - 主キー)
    *   `content` (TEXT - 全文インデックス)
    *   `updated_at` (DATETIME)
    *   `hash` (TEXT - 差分比較用)

### 4.2. 模擬ビジネスデータベース (`business.db` - SQLite)
*   **テーブルおよびインデックス設計**:
    *   `customers` (顧客マスタ): `customer_id` (PK)
    *   `products` (商品マスタ): `product_id` (PK)
    *   `orders` (受注ヘッダー): `order_id` (PK), `customer_id` (FK), `order_date`
        *   *インデックス*: `idx_orders_customer_id`, `idx_orders_order_date`
    *   `order_items` (受注明細): `item_id` (PK), `order_id` (FK), `product_id` (FK)
        *   *インデックス*: `idx_order_items_order_id`, `idx_order_items_product_id`
    *   `accounts` (勘定科目マスタ): `account_code` (PK)
    *   `journals` (仕訳ヘッダー): `journal_id` (PK), `entry_date`
        *   *インデックス*: `idx_journals_entry_date`
    *   `journal_lines` (仕訳明細): `line_id` (PK), `journal_id` (FK), `account_code` (FK)
        *   *インデックス*: `idx_journal_lines_journal_id`, `idx_journal_lines_account_code`
