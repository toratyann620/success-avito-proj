## ページ 6：セキュリティと信頼性

### 閉域網環境の構築
- 入力データは、外部AIサービスの学習に利用されません。
- 機密情報を外部送信しないローカル実行構成とすることで、情報漏えいリスクの低減を図っています。
- 本アプリケーションは **Docker Desktop 上のコンテナ環境** で動作し、実行環境を分離することで、端末上の他アプリケーションへの影響を最小限にする構成としています。

### ハルシネーション抑制
- 承認済み文書や関連資料をRAGで参照しながら回答を生成することで、根拠に基づく回答を優先し、誤回答リスクの低減を図ります。

## ページ 7：生成シーケンス ── Open Notebook をRAG基盤とした資料生成フロー

### 処理フロー

| ステップ | 内容 |
|---------|------|
| ① 要望受付 | 利用者がやりたいことを音声入力 |
| ② PC内部検索 | ローカルPC内の pptx / docx / xlsx / pdf を検索 |
| ③ ファイルサーバ検索 | 社内共有フォルダから関連資料を検索・取り込み。必要があればDB検索 |
| ④ WEB検索（指示時） | 指示があった場合にWEB情報から関連資料を検索・取り込み |
| ⑤ 内部資料生成 | 見積書・社内報告書 → 内部情報のみを根拠に生成 |
| ⑥ 外部提供用資料生成 | 提案書等 → WEB情報も参照 |

### 中核処理基盤

**1. Open Notebook（RAGエンジン）**
- 文書取り込み
- 全文検索 / ベクトル検索
- ノート管理
- RAG対話

**2. Ollama（オラマ：ローカルAIプラットフォーム）**
- ローカルLLM実行基盤

**3. Gemma（ジェマ：ローカルLLM）**
- 要望解釈 / 要約 / 抽出 / 文書ひな形生成

### 運用ルール

| モード | 参照範囲 |
|-------|---------|
| 内部機密文書モード | PC内資料 + ファイルサーバ + DB |
| 提案書モード | PC内資料 + ファイルサーバ + 外部公開情報（WEB検索指示時のみ） |

> ※ 社内情報と外部情報は出典を分離  
> ※ DB連携は段階的提供

## ページ 9：全体アーキテクチャ Step1.（業務DB連携なし）── PC内完全ローカル動作

### ① ユーザー入力
- やりたいことを自然言語で入力（Whisper（ローカル音声認識）等）
- **例：** "A部品を500個発注したい。前回の承認済み見積書を参考に、今回用の見積書をExcelで作成して"

### ② ローカルLLM制御層：オーケストレーション層（指揮）
- ユーザー要求を解析し、処理計画立案（RAG検索 / 出力形式決定）
- RAG検索への問い合わせ生成、外部検索の要否判定
- 回答ドラフトや各種文書の骨子を生成

**使用モデル：** Ollama + Gemma3:12b（LLM）
- Ollama：ローカルLLM実行基盤
- Gemma3:12b：要求解釈・文書生成を担う日本語対応LLM

### ③ Open Notebook（RAGエンジン）

**機能：** 全文検索 + ベクトル検索 · 出典付き回答 · ノート管理 · Docker動作

- 承認済み、提出済み、完成版文書を索引化
- 全文検索・ベクトル検索で関連資料を参照
- 出典付き回答により生成根拠を追跡可能
- ノート単位で案件別・テーマ別に知識を整理

**③-1. PC内部検索クローラー**
- watchdog + pathlib、SQLite FTS5
- watchdog でファイル更新を検知し、pathlib で拡張子や保存先を判定して、必要なら内容をSQLite DBに取り込んでFTS5で全文検索する

**③-2. ファイルサーバフォルダインデクサ【RAG】**
- ファイルサーバフォルダを対象に索引化
- 承認済み・提出済み・完成版だけが入る保管先フォルダを用意し、そのフォルダ群をOpen Notebookの参照対象にする
- 必要に応じて、社内利用許可済みクラウドストレージにも拡張可能

| コンポーネント | 役割 |
|-------------|------|
| LlamaIndex | 集めたファイルをLLMで使いやすい形に加工し、検索・RAG・回答に使う仕組み |
| SQLite FTS5 | 全文検索用の軽量検索エンジン。SQLite = 軽量な組み込み型DB、FTS5 = SQLiteの全文検索機能 |

**③-3. 外部Web検索（指定時）**
- Brave Search API【外部検索指示時のみ】
- 機密情報のない資料作成時のみ使用
  1. Braveで候補URLを20件取得
  2. 官公庁・企業公式・調査会社を優先
  3. 本文抽出（各ページを3〜5行で要約）
  4. 要約元のURLとタイトルを保持（回答時に引用元を表示）
  5. 再利用価値が高いものだけRAGへ格納

### ④ 文書生成層（文書出力形式決定）

| 形式 | ライブラリ | 用途 |
|-----|----------|------|
| Excel | openpyxl | 見積書 |
| Word | python-docx | 報告書 |
| PowerPoint | python-pptx | 提案書 |

### ⑤ 人が編集・最終化
- 内容妥当性、表現、機密性を確認
- 必要に応じて修正し、完成版として確定

> **すべての処理はPC内で完結 — 社内文書のクラウド送信ゼロで、安全に文書たたき台を高速生成**

## ページ 11：全体アーキテクチャ Step2.（業務DB連携あり：会計＆販売DB）

### 既存システム（DB連携なし）側

**① ユーザー入力**
- 自然言語で要望を入力：例 "前月比較業績レポートを作成してください"

**② Ollama + Gemma3:12b（LLM制御部：オーケストレーション層（指揮））**
- 要望理解
- 処理計画立案（RAG検索 / DB照会 / 出力形式決定）
- 自然言語 → 必要な検索条件やSQL生成指示

**③ Open Notebook（RAGエンジン）**
- PC / ファイルサーバのフォルダの過去提案書・報告書・業務資料を検索参照
- 全文検索し、出典付き回答（LlamaIndex、SQLite FTS5等）

**④ 文書生成（ドラフト作成）**
- Word報告書 / PPT提案書 / Excel見積書（openpyxl, python-docx, python-pptx等）

【RAGに取り込む資料】
- 承認済み提案書
- 完成した月次報告書
- 顧客提出済み報告書＆見積書等

---

### DB（会計＆販売）連携モジュール

**⑤ LLM-to-SQL変換エンジン（NL2SQL）**
- 自然言語 → SQL自動生成
- スキーマ/カタログ参照・SQLの最適化

**スキーマ / 業務定義カタログ（事前作成）**
- テーブル定義 / 項目定義 / KPI定義
- 業務用語とDB項目の対応管理
- SQL生成時の意味補正・精度向上

**⑥ SQL制御＆実行**
- SQLチェック（禁止SQL防止等）→ SQL実行
- 件数上限 / タイムアウト制御、高負荷クエリ抑止、外れ値・異常値検知

**⑦ DB接続ブリッジ**
- SQLAlchemy + 読み取り専用接続
- Oracle / PostgreSQL / MySQL / SQL Server 対応
- 接続先制御・認証情報管理・監査ログ記録

**⑧ 会計DB / 販売DB**

| DB | 保有データ |
|----|----------|
| 会計DB | 売上/費用/利益、P/L・B/S |
| 販売DB | 受注/出荷/顧客、商品・地域別 |

**⑨ 帳票生成エンジン**
- Excel（openpyxl）：グラフ・表・KPI
- PPT（python-pptx）：経営会議スライド

**⑩ 監査ログ / 実行履歴**
- 実行SQL・利用者・実行時刻を記録出力

> **【DBは読み取り専用接続】書き込み権限なし / 社外ネットワーク不使用**

## 資料全体像サマリ

### システムの位置づけ
- **スタンドアロン型（クローズド環境）** で動作する、企業向けAI文書検索・作成支援ツール
- 社内情報が一切外部に出ないことを最重要原則としている

### 開発ステップ

| フェーズ | 内容 |
|---------|------|
| **Step1** | PC内完全ローカル動作。業務DBとの連携なし。ソフトウェア費用ゼロ。 |
| **Step2** | 会計DB・販売DBとのNL2SQL連携。段階的拡張。 |
| **最終系** | フェデレーテッドデータマートによる全社DB統合連携 |

### 主要技術スタック

| カテゴリ | 技術 |
|---------|------|
| LLM | Gemma3:12b（Ollama経由、ローカル動作） |
| RAGエンジン | Open Notebook + LlamaIndex |
| 全文検索 | SQLite FTS5 |
| 文書生成 | openpyxl / python-docx / python-pptx |
| 音声認識 | Whisper（OpenAI、ローカル動作） |
| 外部検索 | Brave Search API（指示時のみ） |
| コンテナ | Docker Desktop |
| DB接続 | SQLAlchemy（読み取り専用） |

### ビジネス上のポイント

- **費用：Step1は完全無料**（PCハードウェア・電力コストのみ）
- **セキュリティ：閉域網運用**（クラウド送信ゼロ）
- **対応DB**：Oracle / PostgreSQL / MySQL / SQL Server
- **出力形式**：Excel / Word / PowerPoint
