# AVITO ユーザー操作マニュアル (AVITO II / DB連携・NL2SQL対応版)

本システム「AVITO（アビト）」は、完全ローカルで稼働するAI駆動型ナレッジ検索・文書作成支援プラットフォームです。

---

## 1. システムの起動と停止

本システムはすべて Docker コンテナ上で動作します。ターミナル（コマンドプロンプト）でプロジェクトのルートディレクトリに移動し、以下のコマンドを実行します。

### 起動方法
```bash
docker compose up -d
```
> [!NOTE]
> 初回起動時は、ローカルLLM（Gemma3:12b）の自動ダウンロード（約8GB）が実行されるため、稼働まで10〜30分程度かかる場合があります。

### 停止方法
```bash
docker compose down
```

## 3. チャットUIの使い方

チャットUIでは、社内資料に基づいた高精度なRAG回答を得ることができます。

### 基本操作
1. **テキスト入力**: 通常のチャットツール同様にテキストを入力し、送信します。
2. **回答の根拠表示**: AI의 回答には、参照したローカルドキュメント名とそのスニペット（抜粋）が出典元として明示されます。
3. **デモモード**: LLMのダウンロード完了前や未接続時であっても、ヘッダー右上の「デモモード」をONにすることで、モックデータによる高度なシミュレーション応答や資料生成のインタラクションを確認できます。

## 4. ナレッジ監視フォルダ (RAG) の設定

RAGで参照させたい社内ドキュメント（Word / Excel / PowerPoint / PDF / Text / Markdown）を監視対象フォルダに配置するだけで、AIが自動的に読み込んで検索インデックスを作成します。

### ローカル監視フォルダ
プロジェクトルートにある [watch/](file:///Users/kurokawamutsuo/開発フォルダ/051_AI文書検索作成Proj/watch) ディレクトリ内にファイルを配置してください。数秒で自動検知され、インデクシングが完了します。

### ファイルサーバ（定期スキャン）の設定
ネットワーク経由でマウントしたフォルダ（SMBやNFSなど）を監視する場合、リアルタイム検知が作動しないことがあります。これに対応するため、定期的なスキャン（ポーリング）が標準で有効になっています。
- **設定値**: デフォルトでは5分（300秒）間隔で差分スキャンを実行し、新規追加・修正・削除を自動反映します。
- **カスタマイズ**: `docker-compose.yml` 内の `POLL_INTERVAL_SECONDS` を変更することで、スキャン間隔を秒単位で自由に調整できます。

## 5. 音声入力の使い方

マイクに向かって操作指示を話すだけで、テキスト入力およびアクションを実行できます。

1. チャットUIの入力欄右側にある **「マイクアイコン」** をクリックして録音を開始します。
2. 指示（例: *「最新のセキュリティ規定から、パスワードの変更ルールについて教えて」* など）を話し、再度アイコンをクリックして送信します。
3. ローカルに組み込まれた音声認識AI（Whisper tinyモデル）が、即座に高精度な日本語テキストへ自動変換してチャットに入力します。

## 6. 業務文書の自動生成手順

指示に基づいて、Word、Excel、PowerPoint のテンプレートファイルを自動生成させることができます。

### 作成指示の例
- **Excel**: *「A社向けの見積書を作って」*
- **Word**: *「先週の週次進捗報告書をWordで作成して」*
- **PowerPoint**: *「新製品導入提案のスライドをパワーポイントで作って」*

### ダウンロード方法
AIが文書生成リクエストを検出すると、自動的にバックエンドで物理ファイルを生成します。生成が完了するとチャットバブルの右側に **「ダウンロードボタン（ファイル種別バッジ）」** が表示されますので、クリックしてローカルPCへ保存してください。

## 7. 自然言語でのデータベース照会 (NL2SQL)

AVITOでは、販売データベースや会計データベースの数値データを、専門的なSQLの知識がなくても自然言語で指示するだけで自動取得・可視化できます。

### 7.1. 質問指示の例
- **顧客・販売関連**: *「登録されている顧客の名前と業種を一覧で教えてください。」* *「業種ごとの売上合計（受注額の合計）を教えてください。」*
- **会計・仕訳関連**: *「勘定科目名が『現金預金』の仕訳における、借方金額の合計から貸方金額の合計を引いた現在の残高を計算してください。」*

### 7.2. 画面でのデータ表示（HTMLテーブルビジュアライザ）
データベースからのデータ取得に成功すると、チャットUI内に **グラスモルフィズム調の美しいHTMLテーブル** が動的に挿入され、データレコードが表形式で綺麗に整列して表示されます。

### 7.3. 安全装置（セキュリティブロック）について
ローカルDBを悪意ある操作や誤操作から守るため、データの更新（UPDATE）や削除（DROP）につながる恐れのある指示は、AI側およびデータベース接続の二重の防壁によって自動的に遮断されます。
- **ブロック画面**: 不正な操作を試みると、チャット上に「セキュリティチェック失敗: 禁止された操作キーワードが検出されました」という警告メッセージが表示され、クエリの実行は未然にブロックされます。
- **監査ログ（Audit Trail）の確認**: 管理者は [http://localhost:3101/api/db/audit-logs](http://localhost:3101/api/db/audit-logs) にアクセスすることで、過去のすべてのNL2SQL実行履歴およびセキュリティによるブロック履歴を監査ログとして一覧取得できます。
